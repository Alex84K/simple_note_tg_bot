# simple-note-bot

Записная книжка в Telegram. Присылаешь боту текст / ссылку / фото / файл — он сохраняет
в свою БД. Сам чат с ботом синхронизируется между всеми устройствами средствами Telegram,
а бот добавляет структуру: типы заметок, теги, поиск.

## Стек
- Bun + TypeScript
- [grammY](https://grammy.dev) — Telegram Bot API
- `bun:sqlite` — хранилище (файл `data/notes.db`)
- Файлы/изображения — локальный диск (`data/files`)

## Запуск
1. Создай бота у [@BotFather](https://t.me/BotFather), получи токен.
2. Скопируй конфиг и впиши токен:
   ```sh
   cp .env.example .env
   # отредактируй BOT_TOKEN
   ```
3. Установи зависимости и запусти:
   ```sh
   bun install
   bun run dev    # с авто-перезапуском
   # или
   bun start
   ```

## Использование
- Любое сообщение боту → сохраняется как заметка.
- Ссылка распознаётся автоматически (тип `link`).
- `#теги` в тексте/подписи привязываются к заметке.
- Команды:
  - `/list` — последние заметки
  - `/search <текст>` — поиск по содержимому
  - `/del <id>` — удалить заметку

## Структура
```
src/
  config.ts          # чтение .env
  bot.ts             # точка входа, обработчики grammY
  db/
    db.ts            # инициализация sqlite + схема
    repo.ts          # CRUD-операции
  storage/
    files.ts         # скачивание файлов Telegram на диск
  handlers/
    parse.ts         # извлечение тегов, распознавание ссылок
```

## Заложено на будущее
- `notes.updated_at` и `deleted_at` (soft delete) — основа для будущего веб-клиента и sync.
- Логика отделена от транспорта (`repo` ↔ `bot`) — поверх той же БД можно поднять HTTP API.
